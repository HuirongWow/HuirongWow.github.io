BFC是什么？解决了什么问题？如何创建？

常听到BFC这个词，也知道它是指块级格式作用域，是CSS中很重要的概念。但是对于它到底能解决什么问题，触发它的条件，以及它具体的定义还是比较模糊，查阅了各位大佬的博客，做一篇总结，以备随时温故。

BFC是什么

块格式化上下文（Block Formatting Context，BFC） 是Web页面的可视化CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。 —— [ MDV ]

块级格式化上下文包含创建它的元素内部的所有内容，

块级格式化上下文对浮动定位与清除浮动都很重要。浮动定位和清除浮动时只会应用于同一个BFC内的元素。浮动不会影响其他BFC中元素的布局，同样的，清除浮动只能清除同一BFC内的浮动。margin折叠也只发生于同一BFC中的块级元素之间。

上面这两段话我认为充分体现了BFC的“独立性”，它就像是一个独立的容器，内部有自己的一套规则，同时又不会影响外部或其他BFC。

BFC有哪些规则？

现在我们知道BFC的定义，了解BFC有一些内部规则，那么BFC为何被频繁使用呢？它具体有哪些规则？

BFC的范围

BFC的范围在MDN中的描述，用中文翻译一下是这样的：

一个BFC它包含创建中这个上下文的所有元素，但是不包含创建了新BFC的子元素的内部元素。

换句话说就是一个BFC_1，如果你在它内部新创建了一个BFC_2，那么BFC_1的上下文并不包含BFC_2中的内部元素，它们是相互独立的，即一个元素不能同属于2个BFC。

BFC的一个最重要的效果是，让处在BFC内部的元素于外部的元素相互隔离，使内外元素的定位不会相互影响。这是利用BFC清除浮动所利用的特性，后面会给一个小例子。

那反向思考，如果一个元素可以同时处在两个BFC中，也就是它可以同时影响两个BFC，那就违反了BFC的隔离规则，这个假设就不成立了。

BFC的效果

如上文所说，最明显的效果就是它是一个独立的隔离的空间，不会影响空间外其他元素的作用。

BFC其他的规则

除了具有自己的一套规则。具有独立/隔离性以外，它还：

  1. 内部的盒子在垂直方向按序排列
  2. 同一个BFC中的元素互相影响，两个div的margin可能会发生重叠现象
  3. 每个元素的margin box左边，与容器的border box的左边接触。存在浮动时亦如此。
  4. 计算BFC高度时，考虑BFC包含的所有元素，浮动元素也参与计算。
  5. 浮动盒区域不叠加到BFC上，我理解时不会出现元素环绕现象。

如何创建BFC

BFC的创建方法，此处引用MDN：

  • 根元素()
  • 浮动元素（元素的 float 不是 none）
  • 绝对定位元素（元素的 position 为 absolute 或 fixed）
  • 行内块元素（元素的 display 为 inline-block）
  • 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）
  • 表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）
  • 匿名表格单元格元素（元素的 display为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot的默认属性）或 inline-table）
  • overflow 值不为 visible 的块元素
  • display 值为 flow-root 的元素
  • contain 值为 layout、content或 paint 的元素
  • 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）
  • 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
  • column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

其中最常见的就是overflow:hidden、float:left/right、position:absolute。其实我们在不知道BFC这个概念的时候，会经常使用BFC来做一些事情，比如overflow清除浮动啦，绝对定位啦。每次看到这些属性的时候，就代表该元素已经创建了一个BFC。
